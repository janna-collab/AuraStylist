# AuraStylist Project Architecture Overview

AuraStylist is a full-stack web application designed to be an AI-powered personal stylist. The architecture relies heavily on **Next.js** (for a modern, server-side rendered React interface) and **FastAPI** (for a high-performance Python backend).

Here is a complete breakdown of every directory and file so your team understands how the components interact.

---

## 🖥 Frontend Repository: `aurastylist-frontend`

The frontend uses Next.js 14 structured with its new `"App Router"` which uses the `app/` directory instead of the traditional `pages/` directory. It uses Tailwind CSS for all styling and Lucide-React for crisp vector icons.

### `app/` Directory (Routing & Pages)
In Next.js, whatever folder name exists in `app/` becomes the URL route. The `page.js` inside that folder is the UI rendered at that route.
*   **`layout.js`**: The global wrapper for the * entire * application. We inject our `<StylistChatBot />` component here so it persists across all pages.
*   **`page.js`**: The root landing page (`localhost:3000/`) featuring the bright, animated hero section.
*   **`getting-started/page.js`**: The 3-step onboarding wizard. Handles the photo upload, the manual data form, and displays the final generated Style Report.
*   **`dashboard/page.js` or `style-request/page.js`**: The primary "Style Me" control center where users request new outfits for specific venues.
*   **`gallery/page.js`**: Displays the high-fidelity Virtual Try-On images returned by Amazon Titan. Has buttons to "Shop Look" or "Regenerate".
*   **`shop/page.js`**: The final destination. It displays the automated shopping links (scraped by the backend Playwright script) for real clothing items that match the AI-generated outfit.

### `components/` Directory (Reusable UI)
We break complex UI down into reusable, smaller files instead of keeping them inside `page.js`.
*   **`chat/StylistChatBot.js`**: The floating UI bubble for Nova Sonic/Lite interactions.
*   **`onboarding/`**:
    *   `PhotoUpload.js`: Handles the drag-and-drop or camera input.
    *   `ManualInputsForm.js`: The inputs for Height, Shoe Size, and Preferred Fit.
    *   `ReportResult.js`: Formats the complex JSON generated by Nova Pro into the pretty "Your Style Report" view.
*   **`gallery/OutfitCard.js`**: The individual image container used inside the Gallery grid.

## ⚙️ Backend Repository: `aurastylist-backend`

The backend is a Python FastAPI server. FastAPI is chosen because it is incredibly fast (asynchronous) and natively supports data validation via Pydantic.

### Root Files
*   **`main.py`**: The entry point of the server. This file defines the `app` instance, configures CORS (so the Next.js port 3000 is allowed to talk to it), and registers all the routers.
*   **`requirements.txt`**: Lists all Python dependencies (FastAPI, Uvicorn, Boto3 for AWS, Playwright for scraping, PyMongo).
*   **`.env`**: Stores secret variables like your `AWS_ACCESS_KEY_ID`.

### `routers/` Directory (API Endpoints)
Instead of putting all our API endpoints in `main.py` (which creates a massive, unreadable file), FastAPI lets us group endpoints into "Routers".
*   **`profile.py` (`/api/profile/generate`)**: Receives the user's initial onboarding photo and stats. It calls Nova to generate the master profile.
*   **`style_request.py` (`/api/style/request`)**: Handles the complex logic of distinguishing between styling "myself" vs styling "someone else".
*   **`gallery.py` (`/api/gallery/...`)**: Endpoints involving invoking Amazon Titan to generate images.
*   **`shop.py` (`/api/shop/search`)**: Listens for the "Complete the Look" button click from the frontend and triggers the Playwright scraping agent.
*   **`chat.py` (`/api/chat/...`)**: Handles the text, multimodal (image), and voice streaming requests purely for the `StylistChatBot` component.

### `services/` Directory (Third-Party Integrations)
This is where the actual "heavy lifting" logic resides. The `routers` receive the traffic, but they hand the work off to the `services`.
*   **`nova_service.py`**: The absolute core of the backend. Contains dozens of wrapper functions (like `invoke_titan_image_generation()` or `extract_outfit_components()`) that handle the complex authentication and JSON-formatting required to send correct traffic to the Amazon Bedrock API using `boto3`.
*   **`shopping_agent.py`**: The automated scraping script using Playwright. When called, it literally opens a Chromium browser in the background, navigates to Google, types in a search term, scrapes the first few product results, and returns them to the frontend.

### `core/` Directory (Infrastructure)
*   **`database.py`**: The interface layer for MongoDB. It has functions like `save_user_profile()` or `get_style_request()` so that we aren't writing raw database queries inside our `routers/` or `services/`.
